# Bitcode Need-Synthesis Web Search Agent

This package keeps the retained `web-search` import path, but V26 semantics are
not a generic search product. The active owner is
`bitcodeNeedSynthesisWebSearch`: a discovery-phase support agent that gathers
source-attributed external evidence for Bitcode need synthesis.

The compatibility exports `webSearch`, `quickWebSearch`, `webSearchPrompt`, and
`webSearchStepPrompts` remain available only as carriers for the same Bitcode
purpose. They must not be used to claim canonical need interpretation, proof
closure, mutation authority, delivery selection, Bitcode Exchange ownership, or
Bitcode Terminal ownership.

## Scope

- Collect primary, official, standards, repository, paper, vendor, or commentary
  evidence for a declared Bitcode need or proof gap.
- Preserve URL, snippet, provider, source class, quality score, volatility
  questions, and proof-boundary warnings.
- Support downstream need measurement, proof-gap question formation,
  third-party interface planning, and AssetPack planning.
- Return unresolved gaps rather than inventing closure when source evidence is
  insufficient.

## Non-Scope

- Autonomous crawler product behavior.
- Production search-platform monitoring or enterprise search service ownership.
- Canonical proof generation or final proof acceptance.
- Source mutation, branch writing, PR creation, or delivery mechanism selection.
- Exchange or Terminal product semantics.
