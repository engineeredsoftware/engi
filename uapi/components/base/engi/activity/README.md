# Shared Activity Carriers

This directory holds the shared Bitcode activity vocabulary used to converge retained execution streams, notifications, and later public/personal system usage into one merged-world activity model.

Current active owner:

- `bitcode-activity-model.ts`
  Shared typed Bitcode activity record model plus normalization helpers for retained execution events and notifications.

Fourth-gate rule:
- retained `/executions` must teach `activity` rather than staying an isolated naming island
- notifications are an admitted activity class alongside transactions
- later public and personal system usage may join the same activity family without changing the core shared vocabulary again
