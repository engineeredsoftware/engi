## Field Intelligence

`field-intelligence` was an old-world Engi-era experimental concept for querying speculative production-learning signals through MCP.

It is not part of Bitcode V26.

Why it was removed:
- it depended on incomplete and non-admitted runtime/contracts
- it mixed exploratory intelligence ideas with active Bitcode MCP product claims
- it overstated production readiness and commercial Bitcode scope
- it did not satisfy the current fifth-gate requirement for precise, auditable, fail-closed Bitcode infrastructure

Historical idea summary:
- query code snippets against a hypothetical field-learning network
- analyze files for inferred production patterns and evolution suggestions
- synthesize speculative performance, usage, and recommendation outputs

V26 stance:
- no active Bitcode route, API, MCP surface, proof family, or package boundary may claim `field-intelligence` as part of Bitcode
- if any future Bitcode capability wants related ideas, it must be redesigned from current canon, with explicit data ownership, admission rules, persistence model, proofability, and product purpose

This directory exists only as legacy documentation of the removed idea.
