# Bitcode Execution Work Summary Notes

Status: non-canonical internal note.

## Purpose

Work summaries make Bitcode executions inspectable while they run and after they Finish.

Summary types should describe:
- phase progress,
- Read measurement evidence,
- Read review state,
- fit review status,
- implementation changes,
- validation issues,
- AssetPack output,
- delivery mechanism result,
- settlement/proof references.

## Requirements

- Do not summarize only generic work completion.
- Summaries must identify what was measured, fitted, synthesized, validated, saved, and delivered.
- Final summaries should live under Finish semantics and be rereadable from Exchange history.
